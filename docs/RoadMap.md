# 开发路线规划
## 1. 迁移并改进 爬虫 读取 日K线数据
首先是基于SteamDT的Api和请求网址，爬取日K线数据，并进行解析，然后使用我们自己设计的Api进行包装；其中每日的爬取限制和爬取数据缓存保存在本地db 【done！】
## 2. 拓展开发 爬虫 读取 周K线和时K线数据
这个部分可以先 搁置！ 为什么呢，因为周K线需要市场的稳定 然后作用是分析出未来的趋势；而时K线是需要短线操作的，但是因为这个市场是**t+7**的一个市场，没有办法进行短线操作！所以这两个的功能不是必要功能，可以放在最后阶段作为改进的feature
## 3. 用户数据库
目前设想是两张表：1个记录用户的登录注册信息，包括id,用户名、密码、电子邮箱等等，1个记录主键id（用户的），他追踪的饰品hashName；这部分作为发送报告和用户信息储存的基石 【done！】
## 4. 前端框架
和后端的这些feature进行同步开发，使用vue3框架，css为极客风格，所有的container设计为控制台样式，和个人主站适配 【done！】
## 5. k线图的显示，用户信息显示
显示近期的市场K线图，反应市场的活跃度和健康度；显示用户的基础信息和他们分别选择追踪的饰品
## 6. 预测模型训练
使用常规的LBGM或者LSTM模型进行简单的预测训练分析（一般这些模型的测试集方向正确率再50%-55%之间），可以给AI分析提供额外维度信息 
直接 python models/train_model.py
【done！】
